# git操作
コミットはConventional Commits形式
<type>[(scope)]: <description>

タイプ:
- feat: 新機能
- fix: バグ修正
- docs: ドキュメントの追加
- style: コードのスタイル変更
- refactor: コードのリファクタリング
- test: テストの追加
- chore: ツールの更新
- perf: パフォーマンス改善
- ci: CIの設定変更
- build: ビルドの設定変更
- revert: コミットを取り消す

ルール:
- コミットは意味のある単位でまとめる
